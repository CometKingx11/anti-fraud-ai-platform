### ✅ 2. 角色区分（student / teacher / admin）

**实现位置**: 
- 模型：`app/models/user.py`
- 装饰器：`app/utils/decorators.py`
- 视图：`app/views/admin_views.py`

**角色定义**:
- **student**: 学生用户，只能填写问卷和查看自己的报告
- **teacher**: 教师角色，可以查看所管理学生的报告（待扩展）
- **admin**: 管理员，拥有所有权限

**权限控制装饰器**:
```python
from app.utils.decorators import role_required, roles_allowed

@admin_bp.route('/users')
@login_required
@role_required('admin')  # 只允许管理员访问
def users():
    pass
```

### ✅ 4. 学生权限控制

#### 4.1 只能填写问卷 + 查看自己报告
**实现位置**: 
- `app/views/questionnaire_views.py`
- `app/views/report_views.py`
- `app/utils/decorators.py`

**权限验证**:
```python
@report_bp.route('/')
@login_required
@validate_submission_ownership  # 验证数据所有权
def view():
    pass
```

#### 4.2 防止重复提交
**实现位置**: `app/views/questionnaire_views.py`

**限制规则**:
- 每个用户 24 小时内只能提交一次问卷
- 使用装饰器 `@prevent_duplicate_submission(hours=24)`

**使用方法**:
```python
@questionnaire_bp.route('/submit', methods=['POST'])
@login_required
@prevent_duplicate_submission(hours=24)
def submit():
    pass
```

### ✅ 5. 数据防篡改

#### 5.1 数据完整性校验
**实现位置**: `app/models/submission.py`

**技术实现**:
- 使用 SHA256 生成提交数据的哈希值
- 存储在 `submission_hash` 字段
- 提供 `verify_integrity()` 方法验证数据完整性

**验证方法**:
```python
submission = Submission.query.get(submission_id)
is_valid = submission.verify_integrity()
if not is_valid:
    flash('警告：该报告数据可能已被篡改', 'danger')
```

#### 5.2 IP 地址记录
**字段**: `Submission.ip_address`
**用途**: 记录提交时的 IP 地址，用于审计和追踪

#### 5.3 数据有效性标记
**字段**: `Submission.is_valid`
**用途**: 管理员可标记数据是否有效（无效数据不参与统计）

## 🔒 安全特性详解

### 1. 密码安全
- **算法**: Werkzeug 的 `pbkdf2:sha256`
- **盐值**: 自动生成随机盐
- **存储**: `password_hash` 字段（128 字符）

### 2. 会话安全
- **Session Cookie**: HttpOnly + SameSite=Lax
- **登录超时**: 可配置（默认浏览器会话期间有效）
- **登出清理**: 清除 session 中的敏感数据

### 3. 数据安全
- **完整性校验**: SHA256 哈希
- **防篡改验证**: `verify_integrity()` 方法
- **IP 追踪**: 记录每次提交的 IP 地址

### 4. 权限控制
- **基于角色**: RBAC（Role-Based Access Control）
- **装饰器**: `@role_required`, `@roles_allowed`
- **所有权验证**: `@validate_submission_ownership`

1. **首次使用必须更新数据库**:
