# 链接安全认证功能说明

## ✅ 已实现的安全功能

### 1. CSRF（跨站请求伪造）保护

#### 技术实现
- **Flask-WTF** v1.2.1 - 提供表单安全和验证功能
- **CSRFProtect** - 全局启用 CSRF 令牌验证
- **Token 机制** - 每个表单提交都包含加密的 CSRF 令牌

#### 保护措施
所有 POST 表单都已添加 CSRF token：
```html
<input type="hidden" name="csrf_token" value="{{ csrf_token() }}"/>
```

#### 受保护的页面
1. ✅ 登录页面 (`/auth/login`)
2. ✅ 问卷提交页面 (`/questionnaire/submit`)
3. ✅ 创建用户页面 (`/admin/users/create`)
4. ✅ 编辑用户页面 (`/admin/users/<id>/edit`)
5. ✅ 批量导入页面 (`/admin/users/import`)

---

### 2. 会话安全管理

#### Cookie 安全配置
```python
SESSION_COOKIE_SECURE = True       # 仅允许 HTTPS 传输（生产环境）
SESSION_COOKIE_HTTPONLY = True     # 防止 JavaScript 访问（防 XSS）
SESSION_COOKIE_SAMESITE = 'Lax'    # 防止 CSRF 攻击
PERMANENT_SESSION_LIFETIME = 3600  # 会话有效期 1 小时
```

#### SECRET_KEY 增强
- 从 16 字节增加到 **32 字节**（256 位加密）
- 使用 `secrets.token_hex(32)` 生成强随机密钥
- 支持从环境变量安全加载

---

### 3. CSRF Token 特性

- **有效期**: 1 小时（3600 秒）
- **自动刷新**: 每次表单加载生成新 token
- **SSL 严格模式**: 开发环境放宽限制（`WTF_CSRF_SSL_STRICT = False`）
- **全局保护**: 所有路由默认启用 CSRF 验证

## 🔒 安全攻击防护

### 防护的攻击类型

1. **CSRF 攻击** ✅
   - 攻击者无法伪造用户请求
   - 每个请求都需要有效的 CSRF token
   
2. **XSS 攻击** ✅
   - HttpOnly cookie 防止 JavaScript 窃取 session
   - SameSite 限制跨站请求

3. **Session 劫持** ✅
   - Secure flag 防止中间人攻击
   - 会话超时机制

4. **重放攻击** ✅
   - CSRF token 一次性使用
   - 时间限制防止重复提交

## 📋 部署建议

### 开发环境
```bash

SECRET_KEY=your-secret-key-here
FLASK_ENV=development
```

### 生产环境 **必须** 配置
```bash
# .env 文件配置
SECRET_KEY=<使用强随机生成的 32 字节密钥>
FLASK_ENV=production
DATABASE_URL=mysql://user:pass@localhost/dbname

# 可选但推荐
SESSION_COOKIE_SECURE=True  # 已在代码中设置
PREFERRED_URL_SCHEME=https  # 强制 HTTPS
```

### 生成强 SECRET_KEY
```python
import secrets
print(secrets.token_hex(32))
```

## ⚠️ 注意事项

### 开发环境特殊处理
- `WTF_CSRF_SSL_STRICT = False` - 允许 HTTP 测试
- `SESSION_COOKIE_SECURE = False` (DevelopmentConfig) - 允许非 HTTPS

### 生产环境必须修改
1. ✅ `SESSION_COOKIE_SECURE = True` - 已配置
2. ⚠️ 确保使用 HTTPS 证书
3. ⚠️ 设置强 SECRET_KEY（不要使用默认值）
4. ⚠️ 定期更换密钥

## 🧪 测试 CSRF 保护

### 测试方法
1. 尝试不使用 CSRF token 提交表单
   ```bash
   curl -X POST http://localhost:5000/auth/login \
        -d "student_id=12345678&password=admin123"
   ```

2. 正确的提交方式
   ```bash

   ```
